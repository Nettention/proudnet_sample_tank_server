Demo Build WebGL Game

Rasakan ProudNet Tank Game langsung di browser Anda! Ini adalah build WebGL langsung dari klien Unity yang terhubung ke server multiplayer nyata yang di-hosting di AWS.

Fitur Game yang Didemonstrasikan:

  • Multiplayer Real-time: Terhubung dan bermain dengan pengguna lain yang mengunjungi halaman ini
  • Jaringan ProudNet: Rasakan jaringan berbasis WebSocket dalam aksi
  • Pertempuran Tank: Bergerak dengan WASD, bidik dengan mouse, tembak dengan klik/spasi
  • Chat P2P: Tekan Enter untuk chat dengan pemain lain
  • Permainan Lintas Platform: Pemain dari build standalone dapat bergabung dalam game yang sama

Detail Teknis:

  • Dibangun dengan Unity 6 dan dikompilasi ke WebGL
  • Menggunakan transport WebSocket ProudNet untuk kompatibilitas browser
  • Di-hosting di AWS S3 dengan CloudFront CDN
  • Backend server berjalan di AWS EC2 dengan Docker
  • Codebase yang sama dengan proyek Unity yang dapat diunduh

Langkah 3: Konfigurasi Pengaturan Jaringan

Atur konfigurasi jaringan dasar untuk game tank:

Buat Konfigurasi Jaringan:

  1. Klik kanan di jendela Project
  2. Pilih Create > ProudNet > Network Config
  3. Beri nama TankNetworkConfig
  4. Konfigurasi pengaturan berikut:
    • Server IP: 127.0.0.1 (localhost untuk pengujian)
    • Server Port: 33334 (sesuai dengan port tank server)

Detail Konfigurasi Jaringan:

Pengaturan Koneksi TankNetworkConfig:

  • Server IP: 127.0.0.1
  • TCP Port: 33334 (untuk build standalone)
  • WebSocket Port: 33335 (untuk build WebGL, hanya didukung oleh server C++)

Catatan: Kita akan menggunakan port 33334 untuk build standalone pada awalnya. Nanti untuk build WebGL, kita akan menggunakan port WebSocket 33335 yang hanya tersedia dalam implementasi server C++ (server C# tidak mendukung koneksi WebSocket).

3. Integrasi Multiplayer Tank

Langkah 1: Gambaran Umum Skrip Jaringan

Klien Unity tank menggunakan sistem jaringan komprehensif yang dibangun di atas ProudNet RMI (Remote Method Invocation). Berikut adalah struktur skrip yang sebenarnya:

Skrip Jaringan Inti:

Assets/
â”œâ”€â”€ Scripts/Network/
â”‚   â”œâ”€â”€ GameNetworkManager.cs     # Manajer jaringan utama (Singleton)
â”‚   â””â”€â”€ NetworkTank.cs           # Sinkronisasi jaringan tank individual
â”œâ”€â”€ Proundnet_Demo/ProudNetScripts/
â”‚   â”œâ”€â”€ Common/
â”‚   â”‚   â””â”€â”€ Vars.cs              # Versi protokol dan konfigurasi server
â”‚   â””â”€â”€ Generated/               # Dibuat otomatis dari Tank.PIDL
â”‚       â”œâ”€â”€ Tank_common.cs       # ID pesan RMI dan konstanta
â”‚       â”œâ”€â”€ Tank_proxy.cs        # Panggilan RMI klien-ke-server
â”‚       â””â”€â”€ Tank_stub.cs         # Handler RMI server-ke-klien

GameNetworkManager.cs - Fitur Utama:

  • Manajemen Koneksi Server: Menangani koneksi ke tank server (TCP/WebSocket)
  • Manajemen Grup P2P: Mengelola grup komunikasi peer-to-peer
  • Siklus Hidup Tank: Memunculkan/menghancurkan tank lokal dan remote
  • Komunikasi RMI: Mengirim/menerima gerakan tank, penembakan, pembaruan kesehatan
  • Sistem Chat: Pesan P2P dengan pemain lain
  • Dukungan Multi-platform: Pemilihan otomatis TCP (standalone) vs WebSocket (WebGL)

NetworkTank.cs - Fitur Tank Individual:

  • Interpolasi Posisi: Sinkronisasi gerakan yang halus untuk tank remote
  • Kontrol Pemain Lokal: Gerakan WASD dan penembakan untuk tank Anda
  • Rendering Pemain Remote: Menampilkan tank pemain lain dengan data jaringan
  • Sinkronisasi Kesehatan: Pembaruan kesehatan real-time di semua klien
  • Sinkronisasi Proyektil: Menangani penembakan peluru dan efek dampak
  • Deteksi Tabrakan: Melacak kerusakan dari ledakan peluru

Kelas RMI yang Dihasilkan (dari Tank.PIDL):

  • Tank_proxy.cs: Metode untuk mengirim data ke server (SendMove, SendFire, SendTankHealthUpdated)
  • Tank_stub.cs: Handler untuk menerima data dari server (OnTankPositionUpdated, OnSpawnBullet, OnPlayerJoined)
  • Tank_common.cs: ID pesan dan konstanta RMI yang dibagikan antara klien dan server

Catatan Penting: File dalam folder Generated/ adalah file yang persis sama yang dihasilkan selama implementasi server C# dalam dokumentasi sebelumnya. Ini memastikan kompatibilitas protokol 100% antara server dan klien Unity. Klien Unity menggunakan kembali kode jaringan yang dihasilkan PIDL yang identik.

Fitur Jaringan Utama yang Diimplementasikan:

  • Sinkronisasi Gerakan Real-time: Pembaruan posisi 20Hz dengan interpolasi
  • Sinkronisasi Proyektil: Peluru yang ditembakkan pemain lain muncul secara instan
  • Manajemen Kesehatan: Kerusakan dan penyembuhan disinkronkan di semua klien
  • Sistem Chat P2P: Pesan langsung antar pemain dalam grup yang sama
  • Pemilihan Tipe Tank: Model tank yang berbeda disinkronkan di semua klien
  • Koneksi Ulang Otomatis: Menangani pemutusan jaringan dengan elegan

Pengujian WebGL

Uji versi web dari klien tank:

Catatan: Ingat bahwa jaringan ProudNet tidak berfungsi dalam mode play Unity Editor ketika platform WebGL dipilih. Anda harus menyelesaikan proses build penuh untuk menguji fitur jaringan.

Proses Build:

  1. Klik "Build" di Build Settings
  2. Buat atau pilih folder untuk folder output: contoh- WebGL_Build/
  3. Tunggu penyelesaian build (mungkin memakan waktu beberapa menit)

Pengaturan Web Server Lokal:

Penting: Build WebGL harus di-host pada web server dan tidak dapat dimainkan dengan membuka file HTML secara langsung secara lokal.

Metode 1: Python HTTP Server

# Navigasi ke folder build WebGL
cd WebGL_Build/

# Python 3 (direkomendasikan)
python -m http.server 8000

# Python 2 (jika Python 3 tidak tersedia)
python -m SimpleHTTPServer 8000

# Contoh port kustom
python -m http.server 3000

Metode 2: VS Code Live Server Extension

  1. Install Extension:
    • Buka VS Code
    • Pergi ke Extensions (Ctrl+Shift+X)
    • Cari "Live Server" by Ritwick Dey
    • Klik "Install"
  2. Host WebGL Build: 
    1. Buka folder WebGL_Build/ di VS Code
2. Klik kanan pada index.html

Pemecahan Masalah

Masalah Umum dan Solusi:

Masalah Koneksi

  • "Gagal terhubung ke server"
    • Verifikasi server tank berjalan dan dapat diakses
    • Periksa pengaturan firewall untuk port 33334-33335
    • Konfirmasi alamat IP server di pengaturan NetworkManager

Masalah Integrasi ProudNet

  • "ProudNet DLL tidak ditemukan"
    • Verifikasi DLL ada di folder Assets/Plugins/
    • Periksa pengaturan platform untuk setiap file DLL
    • Restart Unity Editor setelah perubahan DLL

Masalah Build

  • "Build gagal dengan error jaringan"
    • Pastikan semua script ProudNet dikompilasi tanpa error
    • Periksa Unity Console untuk pesan error spesifik
    • Verifikasi pengaturan platform untuk DLL ProudNet di folder Plugins:
      • Windows: x86_64 diaktifkan untuk build standalone
      • WebGL: platform WebGL diaktifkan untuk build browser

Masalah WebGL

  • "Koneksi WebSocket gagal"
    • Verifikasi server tank mendukung WebSocket (port 33335)
    • Periksa konsol browser untuk pesan error spesifik
    • Pastikan header CORS dikonfigurasi dengan benar di server
  • "Jaringan tidak berfungsi di Unity Editor dengan platform WebGL dipilih"
    • Perilaku yang Diharapkan: ProudNet tidak mendukung mode play in-editor untuk platform WebGL
    • Solusi: Build proyek sebagai WebGL dan uji di web browser
    • Alternatif: Beralih kembali ke platform Windows/Mac/Linux Standalone untuk pengujian editor

Alat Debugging:

  • Unity Console: Koneksi jaringan dan pesan error
  • Server Console: Log koneksi klien dan status game
  • Network Profiler: Analisis performa jaringan bawaan Unity
  • ProudNet Monitor: Statistik jaringan real-time (jika tersedia)
