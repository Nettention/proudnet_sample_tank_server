前置条件

在构建此项目之前，您必须在与tank_server项目文件夹相同级别安装ProudNet文件夹。

所需目录结构

project_root/
├── tank_server/          # This project
│   ├── Common/
│   ├── Server/
│   ├── Client/
│   └── Server_CPP/
└── ProudNet/            # Required: Files from ProudNet installation
    ├── doc/             # ProudNet documentation
    ├── include/         # ProudNet header files
    ├── lib/             # ProudNet library files
    ├── Sample/          # ProudNet sample projects
    └── util/            # ProudNet utilities

ProudNet安装

  1. 从官方网站下载并安装ProudNet
  2. 确保ProudNet文件夹包含以下子目录：
    • doc/ - 文档文件
    • include/ - C++开发的头文件
    • lib/ - .NET和C++的库文件
    • Sample/ - 示例项目
    • util/ - 实用工具和许可证认证

项目结构

📁 Common

包含服务器和客户端实现共同使用的共享资源。

内容：

  • PIDL文件 - 定义RMI（远程方法调用）函数的协议接口定义语言文件
  • 批处理文件 - 将PIDL文件编译为C#和C++存根/代理类的脚本
  • Vars.cs/Vars.h - 共享配置变量（服务器端口、协议版本等）

关键文件：

  • Tank.PIDL - 坦克游戏协议定义
  • CompilePIDL_cs.bat - PIDL编译脚本（C#）
  • CompilePIDL_cpp.bat - PIDL编译脚本（C++）
  • Vars.cs - C#配置
  • Vars.h - C++配置

🖥️ Server

坦克游戏服务器的C#实现。

功能：

  • 实时多人坦克游戏服务器
  • 坦克移动、射击和生命值管理
  • 玩家连接/断开连接处理
  • 高效网络的P2P组管理
  • 服务器控制台命令（伤害、治疗、重生、状态）
  • Windows/.NET兼容

关键文件：

  • TankServer.cs - 主服务器实现
  • TankServer.csproj - .NET项目文件
  • build.bat - 带DLL复制的构建脚本

🎮 Client

坦克游戏的C#控制台客户端。

功能：

  • 坦克游戏客户端的控制台界面
  • 坦克移动和射击控制
  • 与服务器的实时通信
  • P2P消息支持
  • Windows/.NET兼容

关键文件：

  • TankClient.cs - 主客户端实现
  • TankClient.csproj - .NET项目文件
  • build.bat - 带DLL复制的构建脚本

🐧 Server_CPP

支持Linux的坦克游戏服务器C++实现。

功能：

  • 跨平台坦克游戏服务器（Windows/Linux）
  • Docker容器化支持
  • CMake构建系统
  • 与C#版本相同的游戏逻辑
  • Linux生产部署就绪

关键文件：

  • src/TankServer.cpp - 主C++服务器实现
  • CMakeLists.txt - CMake配置
  • Dockerfile - Docker容器配置
  • docker-compose.yml - Docker Compose设置
  • run.bat - Windows执行脚本

构建选项：

  • Windows：使用CMake的Visual Studio
  • Linux：使用CMake的GCC
  • Docker：容器化部署

开发说明

  • C#版本需要.NET 9.0
  • C++版本需要C++17标准
  • 两个版本通过PIDL文件共享相同的协议定义
  • Docker部署支持自动ProudNet库检测和复制

开始使用

前置条件

  1. 从您的ProudNet安装位置复制ProudNet文件，在与tank_server/相同级别创建ProudNet/目录
  2. 验证ProudNet/util/PIDL.exe存在以进行协议编译

1. C#服务器实现

步骤1：理解协议定义（Tank.PIDL）

Tank.PIDL文件定义了服务器和客户端之间的通信协议：

// Tank.PIDL - Protocol Interface Definition Language
global Tank 2000 // Client-Server and Server-Client RMI, first message ID = 2000
{
    // Client to Server functions
    SendMove([in] float posX, [in] float posY, [in] float direction);
    SendFire([in] int shooterId, [in] float direction, [in] float launchForce, ...);
    SendTankType([in] int tankType);
    
    // Server to Client functions  
    OnPlayerJoined([in] int clientId, [in] float posX, [in] float posY, [in] int tankType);
    OnTankPositionUpdated([in] int clientId, [in] float posX, [in] float posY, [in] float direction);
    OnSpawnBullet([in] int clientId, [in] int shooterId, ...);
    // ... more RMI functions
}

此PIDL文件定义了：

  • RMI函数 从客户端发送到服务器（SendMove、SendFire等）
  • RMI函数 从服务器发送到客户端（OnPlayerJoined、OnTankPositionUpdated等）
  • P2P消息 用于直接客户端通信

步骤2：生成C#存根和代理类

运行PIDL编译器生成C#网络代码：

cd tank_server/Common
CompilePIDL_cs.bat

预期输出：

PIDL compile complete!

此脚本将：

  • 执行ProudNet/util/PIDL.exe -cs Tank.PIDL -outdir ../Server
  • 执行ProudNet/util/PIDL.exe -cs Tank.PIDL -outdir ../Client
  • 在Server和Client目录中生成文件：
    • Tank_stub.cs - 处理传入的RMI调用
    • Tank_proxy.cs - 发送传出的RMI调用
    • Tank_common.cs - 通用定义和常量

2. C#客户端实现

步骤1：共享协议理解

客户端使用与服务器相同的Tank.PIDL文件，确保完美的协议兼容性：

  • 相同的RMI定义：服务器和客户端都理解相同的函数签名
  • 共享存根/代理类：CompilePIDL_cs.bat脚本为两个项目生成相同的网络代码
  • 协议同步：对Tank.PIDL的任何更改在重新编译时会自动更新服务器和客户端

这种方法保证：

  • 服务器和客户端始终使用兼容的网络协议
  • RMI函数调用是类型安全的，并在编译时验证
  • 两个应用程序之间的网络消息格式是一致的

3. C++服务器实现（可选）

C++服务器实现提供与C#服务器完全相同的功能，提供跨平台支持和容器化部署选项。这是一个可选的高级实现，展示了ProudNet的多语言能力。

协议兼容性

由于C#和C++服务器都使用相同的Tank.PIDL文件，它们完全兼容：

  • Same RMI functions: Identical network protocol definitions
  • Client compatibility: C# clients can connect to either C# or C++ servers seamlessly
  • Feature parity: Movement, firing, P2P messaging, and all game mechanics work identically

Key Advantages of C++ Implementation

  • Cross-platform support: Runs on Windows, Linux, and containerized environments
  • Performance optimization: Native C++ performance for high-load scenarios
  • Production deployment: Docker containerization for scalable server infrastructure
  • Linux compatibility: Deploy on Ubuntu/Linux servers without .NET dependencies

  • Tank_common.cpp/.h - Common definitions and constants

Step 2: Build the C++ Server (Windows)

Before building the C++ server, ensure you have the following components installed:

Required Software:

  • Visual Studio 2022 (Community/Professional/Enterprise)

  • CMake 3.10 or higher

Required Visual Studio Components:

When installing Visual Studio, make sure to include these workloads and components:

  • Desktop development with C++ workload
  • MSVC v143 - VS 2022 C++ x64/x86 build tools (latest)
  • Windows 11 SDK (or Windows 10 SDK)
  • ATL for v143 build tools (x86 & x64) - Required for ProudNet linking
  • CMake tools for C++
  • Git for Windows (optional, for version control)

Important Note about ATL:

The Active Template Library (ATL) is essential for ProudNet linking on Windows. In CMakeLists.txt, you can see it's explicitly linked:

target_link_libraries(TankGameServer

)
