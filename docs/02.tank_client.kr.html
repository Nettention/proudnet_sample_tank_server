단계 3: 네트워크 설정 구성

탱크 게임을 위한 기본 네트워크 구성을 설정합니다:

네트워크 구성 생성:

  1. Project 창에서 마우스 오른쪽 버튼 클릭
  2. Create > ProudNet > Network Config 선택
  3. 이름을 TankNetworkConfig로 지정
  4. 다음 설정을 구성:
    • Server IP: 127.0.0.1 (테스트용 localhost)
    • Server Port: 33334 (탱크 서버 포트와 일치)

네트워크 구성 세부사항:

TankNetworkConfig 연결 설정:

  • 서버 IP: 127.0.0.1
  • TCP 포트: 33334 (standalone 빌드용)
  • WebSocket 포트: 33335 (WebGL 빌드용, C++ 서버에서만 지원)

참고: 처음에는 standalone 빌드를 위해 포트 33334를 사용합니다. 나중에 WebGL 빌드의 경우 C++ 서버 구현에서만 사용 가능한 WebSocket 포트 33335를 사용합니다(C# 서버는 WebSocket 연결을 지원하지 않음).

3. 탱크 멀티플레이어 통합

단계 1: 네트워크 스크립트 개요

탱크 Unity 클라이언트는 ProudNet RMI(Remote Method Invocation)를 기반으로 구축된 포괄적인 네트워킹 시스템을 사용합니다. 실제 스크립트 구조는 다음과 같습니다:

핵심 네트워크 스크립트:

Assets/
├── Scripts/Network/
│   ├── GameNetworkManager.cs     # Main network manager (Singleton)
│   └── NetworkTank.cs           # Individual tank network synchronization
├── Proundnet_Demo/ProudNetScripts/
│   ├── Common/
│   │   └── Vars.cs              # Protocol version and server configuration

│       ├── Tank_common.cs       # RMI message IDs and constants
│       ├── Tank_proxy.cs        # Client-to-server RMI calls
│       └── Tank_stub.cs         # Server-to-client RMI handlers

GameNetworkManager.cs - 주요 기능:

  • 서버 연결 관리: 탱크 서버에 대한 연결 처리 (TCP/WebSocket)
  • P2P 그룹 관리: 피어 투 피어 통신 그룹 관리
  • 탱크 생명주기: 로컬 및 원격 탱크 생성/파괴
  • RMI 통신: 탱크 이동, 발사, 체력 업데이트 송수신
  • 채팅 시스템: 다른 플레이어와의 P2P 메시징
  • 멀티플랫폼 지원: 자동 TCP (standalone) vs WebSocket (WebGL) 선택

NetworkTank.cs - 개별 탱크 기능:

  • 위치 보간: 원격 탱크의 부드러운 이동 동기화
  • 로컬 플레이어 제어: 자신의 탱크에 대한 WASD 이동 및 발사
  • 원격 플레이어 렌더링: 네트워크 데이터로 다른 플레이어의 탱크 표시
  • 체력 동기화: 모든 클라이언트에서 실시간 체력 업데이트
  • 발사체 동기화: 포탄 발사 및 충돌 효과 처리
  • 충돌 감지: 포탄 폭발로 인한 피해 추적

생성된 RMI 클래스 (Tank.PIDL에서):

  • Tank_proxy.cs: 서버로 데이터를 보내는 메서드 (SendMove, SendFire, SendTankHealthUpdated)
  • Tank_stub.cs: 서버로부터 데이터를 받는 핸들러 (OnTankPositionUpdated, OnSpawnBullet, OnPlayerJoined)
  • Tank_common.cs: 클라이언트와 서버 간 공유되는 메시지 ID 및 RMI 상수

중요 참고사항: Generated/ 폴더의 파일들은 이전 문서의 C# 서버 구현 중에 생성된 정확히 동일한 파일입니다. 이는 서버와 Unity 클라이언트 간의 100% 프로토콜 호환성을 보장합니다. Unity 클라이언트는 동일한 PIDL 생성 네트워킹 코드를 재사용합니다.

구현된 주요 네트워크 기능:

  • 실시간 이동 동기화: 보간이 포함된 20Hz 위치 업데이트
  • 발사체 동기화: 다른 플레이어가 발사한 포탄이 즉시 표시
  • 체력 관리: 모든 클라이언트에서 피해 및 치유 동기화
  • P2P 채팅 시스템: 같은 그룹 내 플레이어 간 직접 메시징
  • 탱크 타입 선택: 클라이언트 간 다양한 탱크 모델 동기화
  • 자동 재연결: 네트워크 연결 해제를 우아하게 처리

5. 멀티플레이어 기능 테스트

이동 동기화 테스트

여러 클라이언트에서 실시간 탱크 이동을 테스트합니다:

클라이언트 1 동작:

  • WASD를 사용하여 맵 주변에서 탱크 이동
  • 로컬 클라이언트에서 부드러운 이동 관찰

클라이언트 2 관찰:

  • 클라이언트 1의 탱크가 실시간으로 이동하는 것을 확인
  • 이동이 부드럽고 동기화됨
  • 위치 업데이트가 20Hz로 발생 (구성 가능)

예상 네트워크 메시지:

Client 1 → Server: SendMove(posX: 15.2, posY: 30.1, direction: 45)
Server → Client 2: OnTankPositionUpdated(clientId: 3, posX: 15.2, posY: 30.1, direction: 45)

발사 및 발사체 테스트

포탄 발사 동기화를 테스트합니다:

클라이언트 1 동작:

  • 마우스로 목표물을 향해 조준
  • 왼쪽 마우스 버튼 클릭 또는 스페이스 키를 눌러 발사

클라이언트 2 관찰:

  • 클라이언트 1의 탱크에서 포탄이 생성되는 것을 확인
  • 포탄 궤적이 물리학적으로 정확하게 따라감
  • 충돌 위치에서 폭발 효과 발생
  • 포탄이 클라이언트 2의 탱크에 맞으면 체력 감소

예상 네트워크 메시지:

Client 1 → Server: SendFire(shooterId: 3, direction: 180, launchForce: 25, posX: 15.2, posY: 30.1)
Server → Client 2: OnSpawnBullet(clientId: 3, shooterId: 3, direction: 180, force: 25, ...)
Server → All: OnTankHealthUpdated(clientId: 4, health: 85, maxHealth: 100)

P2P 채팅 시스템 테스트

플레이어 간 실시간 메시징을 테스트합니다:

클라이언트 1 동작:

  • 화면 하단의 채팅 입력 필드 클릭
  • 메시지 입력: "Hello from Client 1!"
  • Enter 키를 눌러 전송

클라이언트 2 관찰:

  • 채팅 로그에 메시지 표시: "Player: Hello from Client 1!"
  • 채팅 로그가 자동으로 스크롤
  • 메시지 타임스탬프 표시

체력 및 피해 테스트

체력 동기화 및 피해 효과를 테스트합니다:

피해 시나리오:

  • 클라이언트 1이 클라이언트 2의 탱크에 포탄 발사
  • 포탄이 맞아서 피해 발생

예상 결과:

  • 클라이언트 2: 체력 감소 (예: 100 → 75)
  • 클라이언트 1: 클라이언트 2의 체력 업데이트 확인
  • 체력 UI: 두 클라이언트 모두 업데이트된 체력 값 표시
  • 시각 효과: 피해 효과 및 파티클 표시

  1. 확장 설치:
    • VS Code 열기
    • 확장으로 이동 (Ctrl+Shift+X)
    • Ritwick Dey의 "Live Server" 검색
    • "설치" 클릭
  2. WebGL 빌드 호스팅: 
    1. VS Code에서 WebGL_Build/ 폴더 열기
2. index.html 우클릭
3. "Open with Live Server" 선택
4. 브라우저가 자동으로 http://127.0.0.1:5500에서 열림

방법 3: Node.js http-server (대안)

# 전역 설치 (일회성 설정)
npm install -g http-server

# 빌드 폴더로 이동하여 서버 실행
cd WebGL_Build/
http-server -p 8000

# CORS 활성화 (필요한 경우)
http-server -p 8000 --cors

접속 URL:

  • Python: http://localhost:8000
  • VS Code Live Server: http://127.0.0.1:5500
  • Node.js: http://localhost:8000

WebGL 브라우저 게임플레이

문제 해결

일반적인 문제 및 해결 방법:

연결 문제

  • "서버에 연결하지 못했습니다"
    • 탱크 서버가 실행 중이고 접근 가능한지 확인
    • 포트 33334-33335에 대한 방화벽 설정 확인
    • NetworkManager 설정에서 서버 IP 주소 확인

ProudNet 통합 문제

  • "ProudNet DLL을 찾을 수 없음"
    • DLL이 Assets/Plugins/ 폴더에 있는지 확인
    • 각 DLL 파일의 플랫폼 설정 확인
    • DLL 변경 후 Unity 에디터 재시작

빌드 문제

  • "네트워킹 오류로 빌드 실패"
    • 모든 ProudNet 스크립트가 오류 없이 컴파일되는지 확인
    • Unity 콘솔에서 구체적인 오류 메시지 확인
    • Plugins 폴더의 ProudNet DLL에 대한 플랫폼 설정 확인:
      • Windows: 독립 실행형 빌드에 대해 x86_64 활성화
      • WebGL: 브라우저 빌드에 대해 WebGL 플랫폼 활성화

WebGL 문제

  • "WebSocket 연결 실패"
    • 탱크 서버가 WebSocket을 지원하는지 확인 (포트 33335)
    • 브라우저 콘솔에서 구체적인 오류 메시지 확인
    • 서버에서 CORS 헤더가 올바르게 구성되어 있는지 확인
  • "WebGL 플랫폼 선택 시 Unity 에디터에서 네트워킹이 작동하지 않음"
    • 예상 동작: ProudNet은 WebGL 플랫폼에 대한 에디터 내 플레이 모드를 지원하지 않음
    • 해결 방법: 프로젝트를 WebGL로 빌드하고 대신 웹 브라우저에서 테스트
    • 대안: 에디터 테스트를 위해 Windows/Mac/Linux 독립 실행형 플랫폼으로 다시 전환

디버깅 도구:

  • Unity Console: 네트워크 연결 및 오류 메시지
  • Server Console: 클라이언트 연결 및 게임 상태 로그
  • Network Profiler: Unity의 내장 네트워크 성능 분석
  • ProudNet Monitor: 실시간 네트워크 통계 (사용 가능한 경우)
